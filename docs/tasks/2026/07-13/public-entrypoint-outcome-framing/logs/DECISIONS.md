# Decisions

**2026-07-13**

- **Background:** 공개 README와 UI metadata 교정에서 문서 역할은 맞았지만 내부 제작 절차가 사용자 결과보다 먼저 나오는 실패가 관찰됐다.
- **Decision:** 공개 진입점에는 확인된 사용자 상황·결과·다음 행동을 먼저 제시하는 한 문장 규칙만 추가한다.
- **Why:** 기존 reader-action 계약을 보존하면서 평가·선택·사용 시작이라는 공개 독자 행동의 모호성만 줄인다.
- **Impact:** 일반 reader-job taxonomy는 넓히지 않고 runbook·API·ADR의 절차 목적과 근거 제한을 forward test로 보호한다.

**2026-07-13**
- 1라운드의 영어 공개 README가 언어 지시 없이 한국어로 바뀌었지만 2라운드의 명시적 영어 README는 영어를 보존했다.
- 이번 변경에서 언어 우선순위 규칙을 추가로 넓히지 않는다.
- 새 공개 진입점 규칙이 언어를 침범했다는 반복 증거가 없고 기존 계약이 대상·저장소·사용자 지시의 언어를 이미 다룬다.
- 언어 지시가 모호한 cross-repository 작업은 별도 재현이 생길 때 독립 과제로 다루며 이번 runtime diff는 outcome framing 한 문장으로 유지한다.
