# 스킬 방향

## 목적

- 이미 선택된 문서나 diff를 최신 지적 기준으로 실제 수정한다.
- 지적을 방어하거나 설명하지 않고, 본문을 더 짧고 정확한 최종 상태로 만든다.

## 핵심 방향

- 최신 사용자 지적을 가장 강한 편집 기준으로 둔다.
- 빼라고 한 내용은 “빼야 한다”라고 쓰지 말고 없앤다.
- 레거시, 예외, 경고, 라우팅은 실제 필요가 확인될 때만 남긴다.
- 문서의 요구 언어와 로캘을 유지한다. 영어뿐 아니라 어떤 비사용 언어든 불필요하게 섞지 않는다.
- 코드 식별자, API 이름, 경로, 명령어, 제품명은 번역 대상으로 보지 않는다.
- 교정 편집에 집중하고, 기준 소스 감사나 작업 상태 저장으로 넓히지 않는다.

## 맡는 질문

- 이 문서가 최신 지적을 실제 본문에 반영했는가.
- 삭제 요청을 경고나 주석으로 되살리지 않았는가.
- 설명이 방어적이거나 장황해지지 않았는가.
- 문서의 요구 언어와 실제 식별자를 구분했는가.

## 범위 밖

- 새 문서 초안 작성.
- 문서 패키지 구조 설계.
- 기준 소스나 문서 소유권 판정.
- 작업 상태, 로그, 인수인계 문서 설계.
- 구현 계획이나 코드 변경 범위 결정.

## 수정 기준

- 반복해서 관찰된 편집 실패만 배포 스킬에 추가한다.
- 스킬 본문은 짧은 실행 계약으로 유지한다.
- 특정 언어권 전용 규칙보다 “요구 언어와 로캘 보존”이라는 일반 규칙을 우선한다.
- 한국어 페어 문서는 영문 기본 스킬의 의미를 자연스럽게 옮기되, 장황한 설명을 늘리지 않는다.
