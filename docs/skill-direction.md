# 스킬 방향

## 목적

- 구체적 대상의 문서를 작성하거나 편집할 때 본문 문장 규율을 맡는다.
- 지적이나 작성 제약을 본문에 직접 반영해 더 짧고 정확한 최종 상태로 만든다.

## 핵심 방향

- 최신 사용자 지적을 최우선 편집 입력으로 둔다.
- 문장 압축 전에 대상 문서의 단일 역할, 핵심 주체, 독자 행동, 책임지는 약속을 먼저 잡는다.
- 이미 구체적으로 요청된 문서 표면을 작성할 때도 사후 교정 메모가 아니라 처음 쓰는 문장에 이 방향을 적용한다.
- 삭제 요청은 실제 삭제로 반영한다.
- 레거시, 예외, 경고, 라우팅은 실제 필요가 확인될 때만 남긴다.
- agent 작업 과정, 판단 보존, 미래 편의, 구현 인접 설명은 대상 문서가 그 역할을 맡을 때만 남긴다.
- 문서의 요구 언어와 로캘을 유지하고, 유지할 비요구 언어 일반 문장은 요구 언어 문장으로 고치며 중복되거나 오래됐거나 여분인 문장은 삭제한다.
- 언어 관련 실패 사례가 특정 언어에서 나왔더라도, 배포 스킬 계약은 “요구된 본문 언어를 지킨다”는 일반 규칙으로 적는다.
- 코드 식별자, API 이름, 경로, 명령어, 제품명은 리터럴로 보존한다.
- 교정 편집에 집중하고, 기준 소스 감사나 작업 상태 저장은 해당 워크플로우로 넘긴다.

## 맡는 질문

- 이 문서가 최신 지적을 실제 본문에 반영했는가.
- 이 문서가 작성 시점부터 요구 언어, 목적, 밀도를 지켰는가.
- 이 문서가 하나의 역할과 독자 행동을 분명히 돕는가.
- 삭제 요청이 실제 삭제로 반영되었는가.
- 설명이 직접적이고 필요한 만큼만 남았는가.
- 문서의 요구 언어와 실제 식별자를 구분했는가.

## 이관 대상

- 새 문서 표면을 만들지 여부 판단.
- 문서 패키지 구조 설계.
- 기준 소스나 문서 소유권 판정.
- 작업 상태, 로그, 인수인계 문서 설계.
- 구현 계획이나 코드 변경 범위 결정.

## 수정 기준

- 반복해서 관찰된 편집 실패만 배포 스킬에 추가한다.
- 스킬 본문은 짧은 실행 계약으로 유지한다.
- 특정 언어권 전용 규칙보다 “요구 언어와 로캘 보존”이라는 일반 규칙을 우선한다. 예시 언어는 관찰 근거이고 적용 범위는 요구 언어 전체다.
- 한국어 페어 문서는 영문 기본 스킬의 의미를 자연스럽게 옮기고 짧은 실행 계약을 유지한다.
