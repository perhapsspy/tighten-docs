# Tighten Docs

## 목적

구체적 대상의 문서를 작성하거나 편집할 때 본문 문장을 잡는다. 최신 지적이나 작성 제약을 반영해 더 짧고 분명한 최종 문장으로 만든다.

이 스킬은 문서 작성 중 문장 규율과 교정 편집용이다. 대상의 문장, 밀도, 요구 언어, 정리만 맡고, 기준 소스 선택, 문서 소유권 결정, 새 문서 표면, 작업 상태, 문서 패키지 설계는 다른 흐름이 맡는다.
대상에 요구된 본문 언어가 있는데 다른 언어의 일반 문장이 섞인 경우, 그 언어 표현 정리도 이 스킬 범위에 포함한다.

## 규칙

- 선택되었거나 이름이 지정되었거나 구체적으로 요청되었거나 방금 만들어진 대상만 다룬다. 대상 문서, 요청 산출물, diff가 불명확하면 대상 표면을 짧게 확인한다.
- 문장 수정 전에 대상의 단일 역할, 핵심 주체, 독자 행동, 문서가 책임지는 약속을 확인한다.
- 최신 지적이나 작성 제약을 최종 문장으로 바꾼다.
- 이미 구체적으로 요청된 대상에 새 문장을 작성할 때는 이 규칙을 먼저 적용한다.
- 거절된 내용, 오래된 내용, 제외된 내용은 삭제하고, 경고나 사유 문구는 대상이 그 목적을 맡을 때만 남긴다.
- 레거시 호환, 대체 경로, 예외, 라우팅 문구는 선택된 대상, 지적, 작성 제약이 현재 필요성을 증명할 때만 추가한다.
- 국소 교정은 국소적으로 유지하고, 사용자가 명시적으로 요구할 때만 새 문서 간 정책으로 넓힌다.
- 방어적 설명, 중복 라우팅, 작업 과정 설명, agent 편의 메모, 사과형 문장, 편집 자체에 대한 메타 설명은 대상이 그 목적을 맡을 때만 남긴다.
- 선택된 대상, 리포, 사용자 지시, 최신 지적, 작성 제약에서 요구된 본문 언어와 로캘을 판단한다. 요구가 분명하면 유지할 일반 문장을 그 언어로 고치고, 다른 언어의 중복되거나 오래됐거나 여분인 일반 문장은 삭제한다. 요구가 불분명하거나 충돌하면 짧게 확인한다.
- 코드 식별자, API 이름, 파일 경로, 명령어, 제품명, 도메인 용어, 인용된 프로토콜 문구, 외부 소유 라벨처럼 원문을 유지해야 하는 리터럴은 그대로 둔다.
- 지적이나 작성 제약과 맞고 여전히 필요한 현재 사실만 유지한다.

## 편집 절차

1. 선택된 대상이나 요청 산출물, 최신 지적이나 작성 제약을 읽는다.
2. 대상의 단일 역할, 핵심 주체, 독자 행동, 문서가 책임지는 약속을 잡는다.
3. 요구된 본문 언어와 로캘, 원문을 유지해야 하는 리터럴을 확인한다.
4. 구체적으로 요청된 대상에 새 문장을 작성할 때는 역할, 독자 행동, 언어, 제약에서 바로 최종 상태 문장을 쓴다.
5. 기존 문장은 그 역할과 독자 행동을 돕는지 기준으로 유지, 교체, 삭제, 압축, 언어 수정 중 하나로 본다.
6. 오래된 표현을 수정된 최종 상태로 바꾼다.
7. 거절된 내용, 방어적 문장, 가상의 레거시 요구, 비요구 언어의 중복되거나 오래됐거나 여분인 일반 문장, agent 편의 메모, 과정 설명을 삭제한다.
8. 유지할 일반 문장이 비요구 언어로 되어 있으면 요구된 본문 언어로 고친다.
9. 반복 설명은 하나의 짧은 문장으로 합친다.
10. 역할이 하나로 읽히는지, 최종 상태 문장인지, 현재 사실만 남았는지, 요구 언어 문장인지, 리뷰 가능한 밀도인지 다시 읽는다.

## 중단 조건

순수 번역이나 지역화, 기준 소스 선택, 문서 소유권 결정, 문서 패키지 재구성, 작업 상태나 인수인계 문서 설계, 새 문서 표면을 만들지 판단하는 일, 구현 계획 변경이 필요하면 멈추고 확인하거나 넘긴다.
