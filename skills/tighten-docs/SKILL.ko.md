# Tighten Docs

## 목적

사용자나 리뷰어의 지적을 받은 뒤, 선택되었거나 이름이 지정되었거나 방금 만들어진 문서나 diff를 그 자리에서 더 짧고 분명하게 고친다. 최신 지적을 설명으로 남기지 않고 최종 문장으로 반영한다.

이 스킬은 교정 편집용이다. 새 문서 작성, 기준 소스 감사, 작업 상태 저장, 문서 패키지 설계는 맡지 않는다.
대상에 요구된 본문 언어가 있는데 다른 언어의 불필요한 일반 문장이 섞인 경우, 그 언어 표현 정리도 이 스킬 범위에 포함한다.

## 규칙

- 선택된 대상만 고친다. 대상 문서나 diff가 불명확하면 새 표면을 만들지 말고 짧게 확인한다.
- 최신 지적은 지적에 대한 설명이 아니라 최종 문장으로 적용한다.
- 거절된 내용, 오래된 내용, 금지된 내용은 경고나 사유로 보존하지 말고 삭제한다.
- 선택된 대상이나 지적이 현재 필요성을 증명하지 않으면 레거시 호환, 대체 경로, 예외, 라우팅 문구를 새로 넣지 않는다.
- 사용자가 명시적으로 요구하지 않으면 국소 교정을 새 문서 간 정책으로 넓히지 않는다.
- 방어적 설명, 중복 라우팅, 작업 과정 설명, 사과형 문장, 편집 자체에 대한 메타 설명을 걷어낸다.
- 선택된 대상, 리포, 사용자 지시, 최신 지적에서 요구된 본문 언어와 로캘을 판단한다. 요구가 분명하면 유지할 일반 문장을 그 언어로 고치고, 다른 언어의 중복되거나 오래됐거나 불필요한 일반 문장은 삭제한다. 요구가 불분명하거나 충돌하면 짧게 확인한다.
- 코드 식별자, API 이름, 파일 경로, 명령어, 제품명, 도메인 용어, 인용된 프로토콜 문구, 외부 소유 라벨처럼 원문을 유지해야 하는 리터럴은 그대로 둔다.
- 지적과 충돌하지 않고 여전히 필요한 현재 사실만 유지한다.

## 편집 절차

1. 선택된 대상과 최신 지적 내용을 읽는다.
2. 요구된 본문 언어와 로캘, 원문을 유지해야 하는 리터럴을 확인한다.
3. 각 문장을 유지, 교체, 삭제, 압축, 언어 수정 중 하나로 본다.
4. 오래된 표현을 수정된 최종 상태로 바꾼다.
5. 거절된 내용, 방어적 문장, 가상의 레거시 요구, 비요구 언어의 중복 일반 문장, 과정 설명을 삭제한다.
6. 유지할 일반 문장이 비요구 언어로 되어 있으면 요구된 본문 언어로 고친다.
7. 반복 설명은 하나의 짧은 문장으로 합친다.
8. 거절된 내용이 되살아나지 않았는지, 가상의 레거시 요구가 없는지, 비요구 언어의 불필요한 일반 문장이 없는지, 리뷰 가능한 밀도인지 다시 읽는다.

## 중단 조건

순수 번역이나 지역화, 기준 소스 선택, 문서 소유권 결정, 문서 패키지 재구성, 작업 상태나 인수인계 문서 설계, 새 문서 작성, 구현 계획 변경이 필요하면 멈추고 확인하거나 넘긴다.
